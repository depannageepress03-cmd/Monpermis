# Guide rapide Render (backend + admin)
# À coller dans le dashboard Render après connexion du repo GitHub.

## Prérequis
1. Pousser TOUT le projet (pas seulement mobile/) vers GitHub :
   https://github.com/depannageepress03-cmd/Monpermis.git
2. Autoriser Render à accéder au repo.

## 1) Web Service — Backend
- Name: monpermis-api
- Root Directory: server
- Runtime: Node
- Build Command: npm install
- Start Command: npm start
- Health Check Path: /api/health

Variables d'environnement (copier depuis server/.env.example) :
NODE_ENV=production
MONGODB_URI=...
JWT_SECRET=...
CLIENT_URL=https://<site-apprenant>.onrender.com
ADMIN_CLIENT_URL=https://<admin>.onrender.com
ALLOWED_ORIGINS=https://<site-apprenant>.onrender.com,https://<admin>.onrender.com
API_PUBLIC_URL=https://<api>.onrender.com
GOOGLE_CLIENT_ID=...
SMTP_HOST / SMTP_PORT / SMTP_SECURE / SMTP_USER / SMTP_PASS / SMTP_FROM
FEDAPAY_PUBLIC_KEY / FEDAPAY_SECRET_KEY / FEDAPAY_ENVIRONMENT=live / FEDAPAY_WEBHOOK_SECRET
ALLOW_ADMIN_REGISTRATION=true
MOBILE_SCHEME=monpermis

## 2) Static Site — Admin
- Name: monpermis-admin
- Root Directory: administration
- Build Command: npm install && npm run build
- Publish Directory: dist
- Env: VITE_API_URL=https://<api>.onrender.com

Rewrite SPA: /* → /index.html

## 3) Après déploiement
- Mettre à jour ALLOWED_ORIGINS / CLIENT_URL / ADMIN_CLIENT_URL / API_PUBLIC_URL avec les URLs finales
- Webhook FedaPay → https://<api>.onrender.com/api/webhooks/fedapay
- Rebuild APK mobile avec EXPO_PUBLIC_API_URL=https://<api>.onrender.com/api
